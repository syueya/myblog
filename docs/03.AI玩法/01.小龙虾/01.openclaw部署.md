## 安装openclaw

使用 CLI 安装器安装

```bash
# 创建openclaw目录
mkdir -p ~/openclaw

# 下载并执行安装脚本（指定路径）
curl -fsSL https://openclaw.ai/install.sh | bash -s -- --prefix ~/openclaw

# 添加 PATH
echo 'export PATH="$HOME/openclaw/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# 验证：能看到具体版本信息，表示安装完成
which openclaw # /root/openclaw/bin/openclaw
openclaw --version # 2026.3.2

```

### 打开仪表板

1、在本地电脑终端使用命令创建到远程 Gateway WS 的本地隧道：`ssh -N -L 18789:127.0.0.1:18789 user@host`

说明：
- `-N`：不执行远程命令,只保持端口转发
- `-L`：表示本地端口转发
- 前面的 `18789`：表示你自己电脑本地监听的端口，这个如果被占用了可以更改
  - 如果你电脑本地 18789 已经被占用，可以换成另外的未被占用的比如18888，后面使用`http://127.0.0.1:18888`打开网页即可
- 后面的 `127.0.0.1:18789`：表示服务器上的目标地址
- `user@host`：服务器的用户名与IP
- 命令相当于你的电脑先用 SSH 连到服务器，顺便把服务器上的 `127.0.0.1:18789` “映射”到你自己电脑的 `127.0.0.1:18789`。  


2、在浏览器里打开 `http://127.0.0.1:18789`访问openclaw页面。

如果浏览器打不开，检查：
- SSH 命令窗口是不是还开着
- 服务器 OpenClaw 是否真的在运行
- 服务器上 `openclaw gateway status` 是否正常
- 你本地端口是不是冲突了


3、如果是首次打开的话，在终端运行 `openclaw dashboard` 获取带 token 的链接（http://127.0.0.1:18789/?token=...）来打开，否则会显示`unauthorized: gateway token missing (open the dashboard URL and paste the token in Control UI settings)`
>- Token 来源：gateway.auth.token（或 OPENCLAW_GATEWAY_TOKEN）
>- UI 在首次加载后存储它，后面就不需要加token了

4、页面如下，在概览页面可以设置语言，虽然我还没发现是什么语言。

![img](./img/openclaw-27.png)


5、如果是使用termius连接服务器，可以直接在Port Forwarding（端口转发）菜单下添加一个Local Forward（本地端口转发）：
- Label: 转发规则的名字/备注名
- Local Port number: 你本地电脑要开哪个端口
- Bind address : 你本地电脑上，这个转发端口监听在哪个地址
- Intermediate host: 通过哪台 SSH 主机作为中转
- Destination address: 到了远程服务器以后，最终要访问哪个地址
- Destination Port number: 远程服务器上目标服务的端口

## 踩过的坑

1、feishu 插件重复

在渠道配置的Install Feishu plugin?选项下，我选了Download from npm (@openclaw/feishu)。然后我之后在运行`openclaw gateway status`有个提示：`Config warnings:\n- plugins.entries.feishu: plugin feishu: duplicate plugin id detected; later plugin may be overridden (/root/.openclaw/extensions/feishu/index.ts)`，告诉我feishu 插件被重复注册了。
![img](./img/openclaw-29.png)

然后我检查配置文件：`cat ~/.openclaw/openclaw.json`，发现plugin里有个npm注册的feishu插件。
![img](./img/openclaw-30.png)

查看插件列表：`openclaw plugins list`，发现有两个 ID 都叫 feishu 的插件。
- 一个是官方自带的：stock:feishu/index.ts
- 一个是本地扩展的：global:feishu/index.ts

我的解决方法：
```
# 确认目录
ls -l ~/.openclaw/extensions  # total 4 drwxr-xr-x 5 root root 4096 Mar  7 14:05 feishu

# 直接删这个目录就行
rm -rf ~/.openclaw/extensions/feishu

# 重新检查目录
ls -l ~/.openclaw/extensions # total 0

# 删除plugin配置里的feishu插件
vi ~/.openclaw/openclaw.json

# 重启服务
openclaw gateway restart

# 查看网关状态，发现飞书重复插件警告已经消失
openclaw gateway status

```


`vi` 删除步骤：
- 打开文件：vi ~/.openclaw/openclaw.json
- 按i进入编辑模式，这时左下角通常会出现 `-- INSERT --`，表示你已经可以编辑了。
- 找到 `plugins`并删除整段 `plugins`，注意"plugins"前面的逗号也要删除
- 按一下Esc退出编辑模式
- 输入:wq然后回车来保存并退出
